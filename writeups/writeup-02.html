SSH Brute Force with Hydra

Credential brute force attack against Metasploitable 2 SSH service

Objective

Attempt to gain SSH access to the victim host by brute forcing credentials using a known username (e.g. msfadmin) and a wordlist. This demonstrates the risk of weak or default credentials and the value of monitoring authentication failures.

Tools used

Hydra

Command used

hydra -l msfadmin -P /usr/share/wordlists/rockyou.txt ssh://192.168.100.20

Expected output

Screenshot: Hydra run showing credentials discovered and number of attempts

Findings

Credentials discovered (placeholder): after a number of attempts, Hydra identified a valid password for the msfadmin account. This grants full shell access to the victim, demonstrating how default or weak passwords enable initial access. Count of attempts and time-to-crack can be documented here.

Wazuh Alert

Rules 5710 / 5716 — Multiple SSH authentication failure alerts. The SIEM detected repeated failed SSH login attempts from the Kali host to the victim, consistent with brute force activity. These rules help blue teams identify credential stuffing and brute force attacks in progress.

Defender Takeaway

Implement account lockout policies and fail2ban (or equivalent) to throttle or block IPs after repeated failures. Monitor auth.log (and forward to SIEM) for SSH failures; correlate by source IP and username. Prefer key-based authentication and disable password auth where possible. Remove or change default credentials and enforce strong password policies.
